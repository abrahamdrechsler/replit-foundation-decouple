PRD: Multi-Level CAD Modeling Prototype (Final)
1. Product Overview

This prototype is a CAD-like architectural modeling environment that operates in real-world units (feet/inches) relative to an origin point (0,0,0).
The system includes:

Multi-level modeling (Level 1 and Level 2; future support for N+ levels)

2D + 3D synchronized canvases

Snapping (grid + geometry)

Room drawing, resizing, moving, deleting

Automatic wall generation with merging and splitting behavior

Floor drawing, edge selection, movement, deletion

Foundation assemblies with offsets and per-edge controls

Wall–foundation clipping logic

Full undo/redo (Ctrl+Z, Ctrl+Y)

JSON-based geometry model with globally unique IDs

Everything should update visually in real time in both 2D and 3D.

3. Geometry Types
3.1 Rooms
Drawing

Always axis-aligned rectangles

Click → click rectangle creation

Shows:

Room boundary line (centerline of future walls)

Wall poche around boundary for visual thickness reference

Wall Generation

Every room generates 4 walls with:

Height: 9 feet

Thickness: 3.5 inches

Extrude upward from level plane

Walls are derived, not independently user-created

Wall Merging Behavior

Walls merge when overlapping or shared, even if partial overlaps

Merging recomputed only after final placement, not during drag

Wall Splitting

When clipping or overlaps require subdivision:

Wall is split into multiple sub-wall segments

Each sub-wall gets its own globally unique ID

Room Editing

Select by clicking inside

Move by dragging (preview only, recompute at end)

Delete via inspector or delete key

Resize in inspector:

Editable width and height

Top-left corner fixed

Adjusting dimensions moves right and bottom edges

3.2 Floors
Drawing

Axis-aligned rectangles

Top surface aligns to level

Extrudes downward 12 inches

Editing

Select by clicking inside

Move by dragging (no regeneration of defaults)

Delete normally

Edge Selection

When a floor is selected:

All four edges display numbered edge handles

Hover highlights each edge

Clicking an edge selects:

The edge itself

Its corresponding foundation assembly (if present)

4. Foundation Assemblies

Each floor edge has its own foundation assembly unless user removes it.

Default Foundation

Thickness: 12 inches

Height: 4 feet

Top aligns with bottom of floor slab

Extrudes downward 4 feet

Exterior faces flush with floor exterior faces

Geometry may visually intersect other geometry unless clipped

Offsets

Each edge supports:

Top Offset

Shifts the top of the foundation wall up or down

Example: +24" raises top 24" above slab bottom (account for slab thickness)

Bottom Offset

Raises or lowers foundation base independently

Negative values dig deeper

Deletion & Re-Adding

Deleting a foundation edge is permanent until user manually adds a new one

Moving a floor does NOT regenerate foundations

Geometry simply translates with the floor

Corners

Edges meet at simple 45° miters

Gaps allowed; no compensation logic needed

6. Wall–Foundation Interaction (Milestone 2)

Each room edge has:

clipByFoundation: false (default)

When enabled:

Only the overlapping volume is removed from the wall

If only part of the wall intersects:

The wall is split into multiple segments

Each segment becomes its own entity

Floor Cutting

Foundations must cut the floor slab, removing all overlapping geometry.

9. Multi-Level Future Requirements (Documented for Forward Compatibility)

Though MVP supports only Level 1 and Level 2, future system requirements:

Unlimited levels

Arbitrary level creation/deletion

Level visibility controls:

Show level geometry

Ghost level geometry

Hide levels

Geometry relationships:

Walls stacking across levels

Floors acting as ceilings for lower levels

Level-based snapping and drawing restrictions

These are not required for MVP, but the design should not block future growth.
